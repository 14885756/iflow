夸迭代分支名称不可和任何特性分支名称冲突
特性分支的完整格式：vmember:feature/1612s1/order-manager
修复分支的完整格式：vmember:hotfix/order-delivery
标签：feature后面的-t会覆盖之前同名的分支，hotfix后面的-t会追加新标签

命令提示符的显示：git用户名@迭代号/项目,如linvanda@1612s1/vmember,换行，$空格

别名：
     sp sprint
     ft feature
     c  create
     t  test
     p  publish
     s  show
     d  delete



切换到某个迭代：
     sprint 1612s1
     sprint 12s1
     sp 12s1
     如果检测到所在的月份和迭代版本号差距超过一个月，则程序启动时会给予警告和建议切换的迭代号

进入项目：
     cd vmember


特性分支：feature、ft
     创建：
          feature c order-manager
          feature c /1612s2/order-manager
          feature c /order-manager
          ft c -y order-manager 如果远程有同名分支，则自动从远程创建
          ft c --no-push|--np order-manager   创建但不推送到远程

     发布到测试：
          ft t 发布当前分支到test分支
          feature t order-manager

     发布到生产:
          feature p -t 检查所有项目的当前迭代的特性分支并列出。请输入要发布的分支，多个用","隔开。或者输入"ex"以输入排除分支列表，或输入"all"发布上列所有分支
          feature p vmember,member-center -t 列出vmember和member-center的所有该迭代的特性分支
          feature p vmember:/weigao,membercenter:order -t 发布vmember的feature/weigao和membercenter的feature/1701s1/order分支
          feature p --continue  继续被中断的发布流程

            分支陈列：
                vmember:
                    [f1] feature/weigao
                    [f2] feature/1701s1/order
                 membercenter:
                    [f3] feature/1701s1/goods-edit

                 ...
                 输入in f1,f2 将要发布的分支
                 或者ex f1,f2... 将要排除的分支
                 all 发布所有分支
                 cancel取消操作

                 如果有-t参数，则确认合并的分支后需要再确认tag信息，可修改tag名称

     删除分支：
          feature d order-manager
          feature d --no-push|--np order-manager     不删除远程分支

     切换到分支：
        ft checkout order-manager
        ft ck order-manager
        ft ck           fetch远程分支


修复分支：hotfix、fix
     创建:
          fix c vmember:order-delivery -m '订单物流bug'
          fix c vmember:order-delivery/kfxt-2278
     其他同特性分支

打标签：
     tag 12s1.00
     tag vmember:12s1.02
     tag -m vmember
     tag -m 打主版本标签
     tag vmember
     tag 打修复版标签



显示分支：
     feature show
     feature show -i
     feature s
     feature s vmember
     fix show
     fix show -i
     fix show -i vmember


分支转换：
     h2f order-delivery
     h2f order-delivery -n
     h2f order-delivery -s 12s1

     h2h order-delivery
     h2h order-delivery -n
     h2h order-delivery -s 12s2

     f2f order-manager
     f2f order-manager -n
     f2f order-manager -s 01s1

     f2h order-manager
     f2h order-manager -n
     f2h order-manager -s 01s1
     

分支重命名（作为对分支转换的辅助使用）
     rename old-branch new-branch 需要都是全名

原生git操作：
     git ...


commit 提交说明
commit -p 提交说明  提交并推送
rename oldbranch newbranch

其他：
     所有的操作都有日志记录
     重要操作需要给予提示并要使用者确认
